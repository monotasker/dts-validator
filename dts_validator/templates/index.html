Welcome to the Distributed Text Services API validator

To validate your API, provide:

  • the base url where your DTS api endpoints can be found
  • sample valid requests to each of the endpoints your implementation supports
  • sample invalid requests to each of the endpoints your implementation supports
    • This validator will then test your API to ensure it meets the requirements of the DTS specification. It will confirm whether:
  • your API is accessible
  • your API exposes the required endpoints
  • for both valid and invalid requests that your API returns
    • the correct response codes
    • json or xml response bodies that satisfy each endpoint's response schema
    • the required html link headers
